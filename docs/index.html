// estrategias de scheduling

    
Cada estrategia imita un algoritmo de scheduler real de un sistema operativo. Todas comparten el mismo movimiento greedy hacia la bandera contraria — lo que cambia es qué thread avanza en cada ronda y cuántos pasos recibe.

    

      

        
SECUENCIAL

        
≈ First-Come First-Served

        
Sin orden fijo, cada jugador avanza un paso hacia la bandera. Simple, sin quantum.

      

      

        
ROUND ROBIN

        
≈ Round Robin

        
El equipo reparte quantum pasos, uno por jugador en orden circular, hasta agotarlo.

      

      

        
SHORTEST

        
≈ Shortest Job First

        
Solo se mueve el jugador más cercano a la bandera contraria, un paso por turno.

      

      

        
USTEDES

        
≈ estrategia propia

        
«Todos tienen su chance»: el más lejano recibe la mayor porción del quantum (dividido en potencias de 2), el más cercano la menor.

      

    

  


  

    
// correr el motor real (C++)

    
La simulación de arriba reproduce la lógica en el navegador. El juego real es C++ nativo con pthreads, semaphore.h y barreras — cloná el repo y compilá:

    
# requiere g++ con pthread (Linux / WSL / macOS)
$ git clone https://github.com/martmallol/multithreading-os
$ cd multithreading-os
$ make            # genera el ejecutable ./juego
$ ./juego         # corre con los inputs de config/
$ make clean      # limpia artefactos de build

    
La estrategia y el quantum se eligen en main.cpp. El tablero, banderas y posiciones iniciales viven en config/config_parameters.csv.
