Every team is a process · every player is a thread · the Game Master arbitrates with semaphores, barriers and mutexes.
First flag captured wins.
how it works & legend
1 · The board
Two teams start in opposite corners of a grid. Each team's goal is to reach the enemy flag before the other does.
2 · Who moves
Teams alternate turns (RED first). On its turn, a team's players step one square toward the enemy flag — blocked by walls and by other players.
3 · The scheduler
The strategy decides which players move and how many steps they get each turn — just like an OS scheduler shares CPU time. Change it below and watch the race change.

// scheduling strategies

    
Each strategy mimics a real OS scheduler algorithm. All share the same greedy movement toward the enemy flag — what changes is which thread advances each round and how many steps it gets.

    

      

        
SEQUENTIAL

        
≈ First-Come First-Served

        
No fixed order — every player advances one step toward the flag. Simple, no quantum.

      

      

        
ROUND ROBIN

        
≈ Round Robin

        
The team spends quantum steps, one per player in circular order, until it runs out.

      

      

        
SHORTEST

        
≈ Shortest Job First

        
Only the player closest to the enemy flag moves, one step per turn.

      

      

        
FAIR SHARE

        
≈ proportional-share scheduler

        
“Everyone gets their chance”: the farthest player receives the biggest share of the quantum (split in powers of 2), the closest the smallest.

      

    

  


  

    
// run the real engine (C++)

    
The simulation above reproduces the logic in the browser. The real game is native C++ with pthreads, semaphores and barriers — clone the repo and build:

    
# requires g++ with pthread (Linux / WSL / macOS)
$ git clone https://github.com/martmallol/multithreading-os
$ cd multithreading-os
$ make            # builds the ./game executable
$ ./game          # runs with the inputs in config/
$ make clean      # removes build artifacts

    
Strategy and quantum are picked in main.cpp. Board size, flags and starting positions live in config/config_parameters.csv.
