// scheduling strategies

    
Each strategy mimics a real OS scheduler algorithm. All share the same greedy movement toward the enemy flag вҖ” what changes is which thread advances each round and how many steps it gets.

    

      

        
SEQUENTIAL

        
вүҲ First-Come First-Served

        
No fixed order вҖ” every player advances one step toward the flag. Simple, no quantum.

      

      

        
ROUND ROBIN

        
вүҲ Round Robin

        
The team spends quantum steps, one per player in circular order, until it runs out.

      

      

        
SHORTEST

        
вүҲ Shortest Job First

        
Only the player closest to the enemy flag moves, one step per turn.

      

      

        
FAIR SHARE

        
вүҲ proportional-share scheduler

        
вҖңEveryone gets their chanceвҖқ: the farthest player receives the biggest share of the quantum (split in powers of 2), the closest the smallest.

      

    

  


  

    
// run the real engine (C++)

    
The simulation above reproduces the logic in the browser. The real game is native C++ with pthreads, semaphores and barriers вҖ” clone the repo and build:

    
# requires g++ with pthread (Linux / WSL / macOS)
$ git clone https://github.com/martmallol/multithreading-os
$ cd multithreading-os
$ make            # builds the ./game executable
$ ./game          # runs with the inputs in config/
$ make clean      # removes build artifacts

    
Strategy and quantum are picked in main.cpp. Board size, flags and starting positions live in config/config_parameters.csv.
